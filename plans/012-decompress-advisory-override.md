## Why this matters

`pnpm audit --prod` reports one **critical** advisory: `decompress <= 4.2.1`
(archive extraction can write files/links outside the target directory â€”
GHSA advisory, patched in >= 4.2.2). It is reachable through the PRODUCTION
dependency chain
`@cornerstonejs/labelmap-interpolation > @itk-wasm/morphological-contour-interpolation > itk-wasm > @itk-wasm/dam > decompress`.
Exposure assessment (from the audit): `decompress` is used by itk-wasm's
Node-side artifact-download path, so browser runtime is unaffected â€” but the
package ships in the published dependency tree, meaning every downstream
consumer's `npm audit` flags critical on install, and any Node/SSR usage of
the chain carries real exposure. The cheap, honest fix is a workspace override
to the patched version (same major, published specifically as the security
patch), plus verifying the package still works.
